## 🚨 The Problem

In the **2018 Kerala floods, 483 people died — 60% within the first 6 hours.**
Not because help didn't exist. Because it *couldn't be found in time.*

When disasters strike:
- 📡 Cell towers **collapse first**
- 🗺️ Roads are **blocked**
- 🧑‍⚕️ Nearby skilled volunteers remain **invisible**
- ⏱️ Average coordination time: **4–6 hours**

---

## 💡 Our Solution

**GuardianNet** is a decentralized, offline-first disaster response coordination platform that:

- Operates at **0% internet connectivity** via Bluetooth Low Energy (BLE) mesh
- Uses **AI matching** to dispatch the right verified responder in **47 seconds**
- Pre-maps and verifies **citizen skills, assets, and emergency resources**
- Learns from every disaster via a **Terrain Memory / Disaster DNA engine**
- Turns **every phone into a network node** — the network grows stronger in chaos

## ✨ Key Features

| Feature | Description |
|---|---|
| 🔵 **BLE Mesh Network** | Data hops phone-to-phone via Bluetooth. No towers. No central server. |
| 🤖 **AI Coordination Engine** | Matches verified responders to needs in 47 seconds |
| 🧠 **Terrain Memory** | Learns escape routes & failure patterns from past disasters |
| 🧩 **Asset-Backed Skill Mapping** | Matches "mountain-climbing doctor with 4x4" to isolated settlement |
| 📉 **Grace-Degrading Connectivity** | 100% → 50% → 0% → Paper backup. Never fully fails. |
| 🧬 **Disaster DNA Engine** | Captures institutional memory — predicts resource gaps before next disaster |
| 👤 **AI Digital Twin per Volunteer** | Tracks fatigue, skill usage, availability — never over-deploys a rescuer |
| 🏅 **Civic XP System** | Volunteers earn government-recognized incentives for preparedness |

## 🏗️ Architecture

```
7-Layer GuardianNet Stack
─────────────────────────────────────────
Layer 1 │ Terrain Intelligence     → 3D GIS, Slope Analysis, Risk Prediction
Layer 2 │ Human Asset Mesh         → Skill Mapping, Asset Registry, Fatigue Monitor
Layer 3 │ Network Communication    → BLE Mesh, WiFi Direct, Offline Alerts
Layer 4 │ AI Coordination Engine   → Matching · Routing · Allocation · Learning  ← CORE
Layer 5 │ Dispatch Engine          → Route Planning, Gap Detection, Resource Allocation
Layer 6 │ Real-Time Dashboard      → Live 3D View, Asset Tracking, Performance Monitoring
Layer 7 │ Learning & Improvement   → Post-Event Analysis, Terrain Memory, System Tuning
─────────────────────────────────────────
Cryptographic Layer: Noise Protocol (XX pattern) · Curve25519 · ChaCha20-Poly1305
```

## 🔐 Security

- End-to-end encrypted sessions via **Noise Protocol Framework (XX pattern)**
- Mutual authentication using **Curve25519** key pairs
- **ChaCha20-Poly1305** AEAD cipher for all transport messages
- Replay attack protection via nonce-based sliding window
- Volunteer identity verified via **Aadhaar + DigiLocker**
